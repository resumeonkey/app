## Tu tarea

Optimiza la sección de skills para que coincida con lo que busca la oferta:

1. **Prioriza** las skills que la oferta menciona explícitamente — ponlas primero
2. **Usa terminología exacta** de la oferta cuando corresponda (ej: si la oferta dice "Microsoft Azure" y el resume dice "Azure", usa "Microsoft Azure")
3. **Reemplaza** skills genéricas o de otro dominio por skills relevantes al puesto — SIEMPRE que estén respaldadas por la experiencia real del candidato (visibles en la sección Experience/Proyectos del contexto del resume)
4. **Mantén el mismo formato visual** exacto: misma cantidad de items, misma estructura de tabla o lista, mismos separadores

## Reglas de honestidad (críticas)

- **OBLIGATORIO — REEMPLAZA SKILLS DE DOMINIO INCORRECTO:** Si la sección de skills actual contiene skills claramente fuera del rol objetivo (ej: "Operations Coordination", "Workforce Scheduling", "Travel Coordination", "Administrative Operations" para un rol QA/Tech/Data), DEBES reemplazarlas por skills del dominio correcto que estén DEMOSTRADAS en la sección Experience del contexto. Esta es una corrección de coherencia, no una invención. El candidato que trabajó 8 años en QA tiene todo el derecho a listar "Quality Assurance" como skill aunque el template anterior diga otra cosa.
- Skills de operaciones/administración que NO corresponden al rol objetivo = REEMPLAZAR siempre, sin excepción.
- **SÍ PUEDES** usar skills respaldadas por la experiencia aunque no aparezcan literalmente en la lista actual de skills.
- **NO PUEDES** agregar skills para las que no haya evidencia en ninguna parte del resume (ni en experience, ni en proyectos, ni en el contexto)
- **NO afirmes** herramientas de testing que no aparecen en el resume: Selenium, Cypress, Playwright, JUnit, TestRail, Robot Framework — solo si están en el contexto
- **NO INFLES** niveles de experticia: si el original dice "Basic Automation Scripting", no lo conviertas en "Advanced Test Automation"
- **NO CAMBIES** el número de items — si la tabla tiene 18 celdas, el output debe tener 18 celdas

## REGLAS ANTI-TRUNCACIÓN (críticas)
- NUNCA cortes un nombre a mitad: "Agile Methodology" → siempre completo, nunca "Agile M"
- El output debe tener EXACTAMENTE la misma cantidad de skills que el original
- Si las skills están en tabla de columnas, mantén el mismo número de filas y columnas
- Si no hay cambios necesarios, devuelve la sección exactamente como está
